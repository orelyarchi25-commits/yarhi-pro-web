# חיבור Firebase (Yarhi Pro Web)

## 1. משתני סביבה

1. העתק `.env.local.example` ל-`.env.local`.
2. מלא את הערכים מ־**Firebase Console → Project settings → Your apps → Config** (אותו בלוק `firebaseConfig`).

אחרי שינוי ב־`.env.local` ייתכן שתצטרך להפעיל מחדש `npm run dev`.

## 2. כללי Firestore (חשוב מאוד)

ברירת המחדל בפרויקט חדש **חוסמת** קריאות/כתיבות מהלקוח.

**בלי פרסום Rules מתאימים – לא תתאפשר התחברות “אמיתית” לאפליקציה:** Auth יכול להצליח, אבל שמירת `users/{uid}` נכשלת ואז נראה כאילו “לא מתחברים”.

1. ב־Console: **Firestore → Rules**.
2. העתק את התוכן מ־`firestore.rules` בפרויקט (או פרסם דרך Firebase CLI).
3. **Publish** (חובה ללחוץ).

הכללים מאפשרים לכל משתמש מחובר לגשת **רק** למסמך `users/{uid}` שלו.

## 3. Authentication

הפעל **Email/Password** תחת **Authentication → Sign-in method**.

ב־**Authentication → Settings → Authorized domains** ודא שמופיעים הדומיינים שלך (למשל `localhost`, דומיין הפרודקשן).

## 4. אבטחה

מפתח ה־`apiKey` בצד הלקוח **ציבורי לפי עיצוב**; ההגנה היא ב־**Firestore Rules**, בשרת (אם תוסיף), וב־Cloud Functions – לא בהסתרת הדף.

## 5. איך יודעים אם החיבור לענן תקין?

בדפי **התחברות / הרשמה / שכחתי סיסמה** מוצגת רצועת סטטוס:

- **אדום – "אין חיבור לענן"** – אין קובץ `.env.local` מלא, או לא הפעלת מחדש את השרת אחרי שינוי. בלי זה האפליקציה **לא** מדברת עם Firebase.
- **ירוק – שם הפרויקט** – משתני הסביבה נטענו; אם עדיין יש שגיאה, בדוק Rules, Firestore, ו־Email/Password (ראה למטה).

## 6. בעיות: «האימייל כבר רשום» / לא מתחבר

1. **Firestore Rules** – אם הרשמה נכשלה באמצע, ייתכן שנוצר משתמש ב-Auth בלי מסמך `users/{uid}`. פרסם Rules כמו בסעיף 2, ואז השתמש ב־**שכחתי סיסמה** לאותו אימייל.
2. **סיסמה** – Firebase לא מגלה אם הסיסמה שגויה או האימייל; תמיד אפשר **איפוס סיסמה** מהאתר.
3. **משתמש שנוצר ב-Console** – חייבים להגדיר סיסמה דרך קישור איפוס מהמייל (אין סיסמה אוטומטית).
4. **מפתח API ב-Google Cloud** – אם הגבלת את ה-API key לפי Referrer, ודא ש־`localhost` ודומיין הפרודקשן מורשים, או הסר הגבלה לבדיקות.

5. **Firestore לא נוצר** – ב־Console צריך פעם אחת ליצור מסד **Firestore** (לא רק Authentication). בלי זה כתיבה ל־`users/{uid}` עלולה להיכשל.
