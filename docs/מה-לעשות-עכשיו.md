# מה לעשות עכשיו? (Yarhi Pro – פשוט)

## אם האתר נתקע על "טוען…"

זה קורה כשיש **Firebase מוגדר** (קובץ `.env.local`) אבל משהו לא מסתנכרן.

1. **פתח את הקונסול בדפדפן** (F12 → Console).  
   אם יש שגיאה אדומה על **Firestore** או **permission** – כנראה **לא פרסמת Rules** (ראה למטה).

2. **בדוק ש-Firestore Rules פורסמו**  
   - Firebase Console → **Firestore Database** → **Rules**  
   - הדבק את התוכן מקובץ `firestore.rules` בפרויקט  
   - לחץ **Publish**

3. **הפעל מחדש את השרת** אחרי שינוי ב-`.env.local`:  
   עצור `npm run dev` (Ctrl+C) והרץ שוב.

4. **רוצה לעבוד בלי ענן זמנית?**  
   שנה את שם הקובץ `.env.local` ל־משהו אחר (למשל `.env.local.off`) והפעל מחדש `npm run dev`.  
   האתר יעבוד במצב **מקומי** (בלי התחברות אמיתית לענן).

## אם אתה כן רוצה Firebase (התחברות אמיתית)

### שלב 1 – משתני סביבה
1. העתק את הקובץ `.env.local.example` לשם **`.env.local`** (בתיקיית הפרויקט).
2. פתח **Firebase Console** → ⚙️ **Project settings** → גלול ל־**Your apps** → **Config**.
3. העתק כל ערך לשורה המתאימה ב־`.env.local` (למשל `apiKey` → `NEXT_PUBLIC_FIREBASE_API_KEY`).

### שלב 2 – כללי Firestore
כמו בסעיף למעלה – **חובה** לפרסם את `firestore.rules`, אחרת האפליקציה לא יכולה לקרוא את `users/{uid}`.

### שלב 3 – התחברות
1. **Authentication** → **Sign-in method** → הפעל **Email/Password**.
2. תחת **Settings** של Authentication → **Authorized domains** – ודא שיש **localhost** (לפיתוח).

### שלב 4 – הרץ שוב
```bash
npm run dev
```

## סיכום קצר

| מה | למה |
|----|-----|
| `.env.local` | מחבר את האתר לפרויקט Firebase שלך |
| `firestore.rules` + Publish | מאפשר למשתמש מחובר לקרוא/לכתוב רק את הפרופיל שלו |
| Email/Password ב-Auth | מאפשר הרשמה והתחברות |

אם אחרי הכל עדיין "טוען" – שלח צילום מסך מה-**Console** (F12).
