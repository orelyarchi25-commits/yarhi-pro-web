# אישור חשבונות קבלנים (מנהל)

## איך זה עובד

- בהרשמה נשמר ב־Firestore: `users/{uid}.accountApproved = false`.
- כל עוד הערך `false` — המשתמש **לא** נכנס לתוכנה (רואה מסך "ממתין לאישור").
- **אי אפשר** לאשר את עצמו מהאפליקציה — רק מנהל ב־Firebase Console (או Admin SDK).

## איך לאשר משתמש

1. **Firebase Console** → **Firestore Database** → אוסף `users`.
2. בחר את המסמך לפי **UID** (מופיע במייל ההתראה או ב־Authentication).
3. ערוך שדה: **`accountApproved`** → ערך בוליאני **`true`** → שמור.
4. המשתמש יכול לרענן את הדף — הגישה נפתחת (או התנתקות והתחברות מחדש).

## משתמשים ישנים (לפני התכונה)

אם אין שדה `accountApproved` במסמך — נחשב **מאושר** (לא ננעלו חשבונות קיימים).

## Rules

חובה **לפרסם** את `firestore.rules` המעודכן בקונסולה — אחרת לקוחות יוכלו לשנות את השדה בעצמם.
